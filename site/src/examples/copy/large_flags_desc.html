You can change the display size of the flags using the CSS variable --iti-flag-width (the height scales automatically). But you cannot change the underlying flag image size without rebuilding it yourself. The flag sprite is pre-built to a specific size (20x15px by default, or 40x30px for retina), so scaling down via the CSS variable works fine, but scaling up past the default looks soft because the sprite is rasterised at a fixed resolution.

To show larger, crisp flags, you need to rebuild the images yourself at a higher resolution. Here's how:

  1. Follow the Contributing guidelines to download the project and get it building (npm run build) without errors.
  2. In scripts/generate-sprite.js, update the TARGET_HEIGHT value — for the demo below we use 18 (so the retina sprite is 36). Whatever height you choose, it must be a multiple of 3 so that the corresponding width is a whole number (flags have a 4:3 aspect ratio).
  3. Run npm run build:img to generate the new flag images and the updated CSS needed to use them.
  4. Open demo.html to check the larger flags render correctly.

You may then want to override some of the CSS variables/styles from src/css/intlTelInput.scss in your own CSS to accommodate the larger flag images — see the overrides used in this demo.

When you're happy, copy the built flag images and CSS file into your own project, and load the standard plugin script as usual.
