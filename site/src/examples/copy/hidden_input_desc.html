If you're handling form submissions using traditional page requests (instead of modern fetch requests), your input values need to contain all the data you want before the request is made. Assuming you want to save phone numbers in standardised international format, which we strongly recommend, this becomes a problem when the user enters their number in national format.

For this situation, the plugin provides the hiddenInput option. It listens for the form submit event and injects a hidden input containing the full international number (and another containing the selected iso2 code, e.g. "gb"), which are submitted along with the rest of your form data.

You can see this in action in the demo below. Select your country, enter a valid phone number in national format (no international dial code), and submit the form. The hidden input full_phone (the full international number) is sent along with the main phone input in the URL parameters.

In this example the form submits to the current URL, so the page reloads with those parameters. When we detect the full_phone parameter on load, we display it below so you can see the full submitted value.
